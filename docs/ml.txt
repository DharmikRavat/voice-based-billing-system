Model 1 — Speech Enhancement (Noise Removal):
  ID:   JorisCos/DCCRNet_Libri1Mix_enhsingle_16k
  Task: Clean kitchen background noise from audio
  When: BEFORE sending audio to Whisper
  How:  POST audio blob → backend → HF Inference API → 
        returns cleaned audio buffer

Model 2 — Speech to Text (Transcription):
  ID:   openai/whisper-medium
  Task: Convert cleaned voice audio → raw text transcript
  When: After noise removal
  How:  POST cleaned audio buffer → HF automaticSpeechRecognition
        → returns { text: "do paneer butter masala aur teen naan" }

Model 3 — Translation (Multilingual → English):
  ID:   Helsinki-NLP/opus-mt-mul-en
  Task: Translate Hindi/Gujarati/Tamil order text → English
  When: After transcription, BEFORE NER
  How:  POST { inputs: transcript_text } → HF translation pipeline
        → returns [{ translation_text: "two paneer butter masala 
           and three naan" }]
  Logic: Always run translation. If input is already English,
         model returns same text — no language detection needed.

Model 4 — Named Entity Recognition (Item Extraction):
  ID:   dslim/bert-base-NER
  Task: Extract food item names and quantities from translated text
  When: After translation
  How:  POST { inputs: english_text } → HF tokenClassification
        → returns array of labeled tokens
  Post-processing: Group consecutive B-ORG/I-ORG/B-PER/I-PER tokens
        into full entity strings. Map entities against MongoDB menu
        collection using case-insensitive regex fuzzy match.
        Extract quantity by scanning tokens BEFORE each entity for
        digits or words: one=1 two=2 three=3 four=4 five=5 six=6
        seven=7 eight=8 nine=9. Default qty=1 if none found.

🔄 COMPLETE REAL DATA FLOW (Step by Step)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1 — Frontend mic capture:
  - User taps mic button
  - Browser MediaRecorder API captures audio as Blob (audio/webm)
  - On stop: send audio Blob as FormData to backend
  - Endpoint: POST /api/ai/process-voice

STEP 2 — Backend noise cleaning:
  - Receive audio file via multer (memoryStorage, no disk writes)
  - Convert audio buffer to correct format
  - POST to HF Inference API:
      model: JorisCos/DCCRNet_Libri1Mix_enhsingle_16k
      data: audioBuffer
  - Receive cleaned audio buffer

STEP 3 — Backend transcription:
  - POST cleaned audio buffer to HF Inference API:
      model: openai/whisper-medium
      data: cleanedAudioBuffer
  - Receive: { text: "raw transcript in any language" }

STEP 4 — Backend translation:
  - POST to HF Inference API:
      model: Helsinki-NLP/opus-mt-mul-en
      inputs: transcript.text
  - Receive: [{ translation_text: "english order text" }]

STEP 5 — Backend NER extraction:
  - POST to HF Inference API:
      model: dslim/bert-base-NER
      inputs: translation_text
  - Receive: array of token classification results
  - Post-process: extract entity strings + quantities
  - Fuzzy match each entity against MongoDB Menu collection:
      db.menu.find({ name: { $regex: entityString, $options: 'i' },
                     available: true })
  - Build resolved items array: 
      [{ menuItemId, name, quantity, unitPrice, totalPrice }]

STEP 6 — Backend billing:
  - subtotal = sum(unitPrice * quantity)
  - taxAmount = subtotal * GST_RATE (from .env)
  - grandTotal = subtotal + taxAmount
  - All values rounded to 2 decimal places

STEP 7 — Backend response to frontend:
  - Return single JSON:
    {
      transcript: "original text",
      translatedText: "english text",
      extractedEntities: [...],
      resolvedItems: [...],
      notFound: [...],
      billing: { subtotal, taxAmount, gstRate, grandTotal }
    }

STEP 8 — Frontend renders real data:
  - Display live transcript from response
  - Display translated text if different
  - Render order items from resolvedItems (NO hardcoded menu)
  - Show billing totals from response
  - User can manually adjust qty +/- (updates totals via 
    recalculation on frontend using unitPrice from response)

STEP 9 — Order submission:
  - User clicks "Generate Bill"
  - Frontend POST /api/orders with:
    { resolvedItems, tableNumber, customerName, 
      paymentMethod, transcript, grandTotal,
      subtotal, taxAmount, gstRate }
  - Backend saves to MongoDB Orders collection
  - Returns saved order with _id and orderNumber

STEP 10 — Invoice display:
  - Frontend renders invoice modal from saved order data
  - All values from MongoDB response — nothing computed on frontend
  - Print button calls window.print()

Export 4 functions:

async enhanceAudio(audioBuffer):
  - client.audioToAudio({
      model: "JorisCos/DCCRNet_Libri1Mix_enhsingle_16k",
      data: audioBuffer
    })
  - Return enhanced audio buffer
  - On HF error: log warning, return ORIGINAL audioBuffer 
    (degraded mode — Whisper still works, just noisier)

async transcribeAudio(audioBuffer):
  - client.automaticSpeechRecognition({
      model: "openai/whisper-medium",
      data: audioBuffer
    })
  - Return result.text (throw if empty)

async translateToEnglish(text):
  - client.translation({
      model: "Helsinki-NLP/opus-mt-mul-en",
      inputs: text
    })
  - Return result[0].translation_text or result.translation_text

async extractEntities(text):
  - client.tokenClassification({
      model: "dslim/bert-base-NER",
      inputs: text
    })
  - Return raw token array

Export parseEntitiesWithQty(tokens, originalText):
  - Group consecutive B-*/I-* tokens into entity phrases
  - For each entity phrase:
      * Scan 3 words BEFORE entity in originalText
      * Look for digit (\d+) or word number (one through nine)
      * Assign quantity (default 1)
  - Also run fallback: split originalText by common separators
    (and, aur, or, comma) and scan each segment for quantity words
  - Return [{ entityText: string, quantity: number }]

Export async matchItemsToMenu(parsedEntities):
  - For each entity:
      * Try exact: Menu.findOne({ normalizedName: entity.toLowerCase() })
      * Try partial: Menu.findOne({ normalizedName: 
          { $regex: entity.toLowerCase(), $options: 'i' } })
      * Try stripped plural: remove trailing 's', retry
      * Try each word individually if multi-word fails
  - Only accept items where available: true
  - Build:
      resolved:    [{ menuItemId, name, quantity, unitPrice, totalPrice }]
      notFound:    [entityText strings with no DB match]
      unavailable: [item names found but available:false]
  - Return { resolved, notFound, unavailable }

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
HuggingFace API errors:
  - 503 Model loading: retry after 20s (max 3 retries)
  - 429 Rate limit: return 429 to frontend with message
  - Network error: return 502 with "AI service unavailable"
  - Empty response: return 422 with "Could not process audio"

Audio errors:
  - Mic permission denied: frontend shows permission guide
  - Empty audio blob: frontend blocks submission
  - Audio too short (<1s): frontend shows "Please speak longer"

MongoDB errors:
  - Connection fail: server exits with clear log
  - Menu item not found: included in notFound[] (not 500)
  - Duplicate order: retry with new orderNumber

Frontend network errors:
  - Show toast with exact error message from backend
  - Never show "Something went wrong" generically
  - Show retry button for AI processing failures
